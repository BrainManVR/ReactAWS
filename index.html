The Foundation

The Mental Shift

Moving from desktop C# or MATLAB scripts to a React/AWS architecture is like moving from a single, well-organized factory to a global, asynchronous supply chain.

Desktop (C# / MATLAB)

  • ■ Stateful: The program stays open in memory. Variables persist until you close the app.
  • ■ Synchronous: Code executes top-to-bottom. If MATLAB is crunching a matrix, the UI freezes unless you explicitly use threads.
  • ■ Monolithic: UI, logic, and file access happen on the same machine.

Web (React / AWS)

  • ■ Stateless Backend: AWS Lambda functions wake up, process data, return it, and die. They forget everything immediately.
  • ■ Asynchronous: Network requests take time. The frontend (React) must keep drawing the UI while waiting for the backend to reply.
  • ■ Distributed: The UI runs on the user's browser. The logic runs on Amazon's servers.
The Frontend

React is just UI Classes

If you know C# Windows Forms or WPF, React is very similar. A "Component" is just a Class or UserControl. "State" is just private variables that, when changed, automatically trigger a UI redraw (like `INotifyPropertyChanged`).

C# WPF / WinForms Mental Model 
public class CounterControl : UserControl 
{
    private int _count = 0;

    // When this changes, UI updates automatically
    public int Count 
    {
        get { return _count; }
        set 
        { 
            _count = value; 
            NotifyPropertyChanged("Count"); 
        }
    }

    public void OnButtonClick(object sender, EventArgs e) 
    {
        // Mutate state
        Count++; 
    }
}
React Component (JavaScript/JSX) 
import { useState } from 'react';

export default function CounterComponent() {
    // Define state (variable) and setter (notifier)
    const [count, setCount] = useState(0);

    const handleButtonClick = () => {
        // Update state (triggers re-render automatically)
        setCount(count + 1);
    };

    // The "Render" method returning HTML (JSX)
    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={handleButtonClick}>
                Add 1
            </button>
        </div>
    );
}
The Backend

AWS: Someone Else's Computer

In MATLAB, you write a function and save variables to a `.mat` file on your hard drive. In AWS, you write a function (Lambda) and save variables to a cloud database (DynamoDB) or cloud drive (S3).

AWS Lambda (The Processor)

MATLAB equivalent: A standalone `.m` script function.
C# equivalent: A specific method in a static class.

Instead of running a server 24/7, Lambda lets you upload a piece of code (in Python, Node.js, C#, etc.). AWS keeps it dormant. When a request comes in (e.g., from your React app), AWS spins up a tiny container, runs your code, returns the answer, and shuts it down. You only pay for the exact milliseconds it runs.

Amazon DynamoDB (The Data)

MATLAB equivalent: A massive Array of Structs.
C# equivalent: A generic Dictionary<string, object>.

It's a NoSQL database. Unlike a strict SQL table, every "row" (item) can have different "columns" (attributes). It is extremely fast and scales infinitely. You look up data using a Primary Key (just like a dictionary key).

Amazon S3 (The File System)

MATLAB equivalent: save('data.mat') to `C:\Data`.
C# equivalent: File.WriteAllBytes().

Simple Storage Service. It's basically a limitless hard drive in the cloud. You use this to store large files: images, PDFs, raw CSV data sets, or even the compiled HTML/JS files that make up your React application!

The Bridge

How they talk: APIs & JSON

In a desktop app, your UI calls a function directly. In web dev, your React UI must send a text message (JSON) over the internet to a router (API Gateway), which hands it to your function (Lambda).

The Asynchronous Problem

In MATLAB, `data = processData(input);` stops the program until it's done. In JavaScript, doing this over the internet would freeze the browser window. We use Promises (the `async/await` keywords, identical to modern C#).

// C# Equivalent: public async Task<Data> GetDataAsync()
const fetchDataFromAWS = async () => {
    try {
        // 'await' pauses THIS function, but keeps UI responsive
        const response = await fetch('https://api-gateway-url.aws.com/prod/getData');
        const jsonData = await response.json();
        
        console.log("Got data: ", jsonData);
        // Update React state here...
    } catch (error) {
        console.error("Network error!", error);
    }
};
